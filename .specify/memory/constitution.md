# Failure Archive Constitution
<!-- This constitution defines the governing principles for the Failure Archive project. -->

## Core Principles

### I. Spec-Driven Development (SDD)
The project strictly adheres to the Speckit workflow. No code shall be implemented without a preceding specification (`spec.md`) and technical plan (`plan.md`). This ensures intentionality and reduces technical debt.

### II. Neural Cyberpunk Aesthetic
The user interface must maintain a high-performance "neural forensic" theme. This includes glassmorphism, high-contrast typography, interactive motion feedback, and a dark-mode-only "classified" aesthetic.

### III. Decoupled Forensic Architecture
The AI Neural Engine (Backend) and the Intelligence Suite (Frontend) must remain strictly decoupled. Communication occurs exclusively via documented REST APIs, ensuring the forensic models can be stress-tested independently.

### IV. Data Determinism & Integrity
Failure data is the project's primary asset. All analysis results must be deterministic and verifiable. System accuracy and data forensic integrity take precedence over raw latency.

### V. Test-First Verification
Every remediation protocol and forensic model update must be validated with automated tests before being merged into the archive.

## Development Workflow
Features follow the sequential path:
1. `/speckit.specify` -> Requirements & UI/UX behavior.
2. `/speckit.plan` -> Technical architecture & data models.
3. `/speckit.tasks` -> Granular implementation steps.
4. `/speckit.implement` -> AI-assisted coding & validation.

## Governance
This constitution is the project's supreme law. Amendments require a formal version bump and justification in the Sync Impact Report.
